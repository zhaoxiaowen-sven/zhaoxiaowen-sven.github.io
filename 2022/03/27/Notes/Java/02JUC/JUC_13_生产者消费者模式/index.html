JUC_13_生产者和消费者模式

生产者 - 消费者模型 Producer-consumer problem 是一个非常经典的多线程并发协作的模型，也是JUC 面试中高频的问题。

简单来说，这个模型是由3个角色：

  • 缓冲区：生产者和消费者的公共存储空间。

  • 生产者：“生产”产品，并把产品放到一个缓冲区，当缓冲区满是阻塞

  • 消费者：“消费”产品，从缓冲区取走产品，当缓冲区为空时阻塞

该模型实现了生产者和消费者之间的解耦和异步。

生产者就只需要关注生产，而不用管消费者的消费行为，更不用等待消费者线程执行完；

消费者也只管消费，不用管生产者是怎么生产的，更不用等着生产者生产。

通常有四种方式可以实现：

  • Object的wait() / notify()方法
  • Lock和Condition的await() / signal()方法
  • BlockingQueue阻塞队列方法
  • Semaphore的acquire()/release()方法

下面来一一介绍下。

一、Object的wait/notify

在使用线程的等待通知机制时，一般都要在 while 循环中调用 wait() 方法。 因为唤醒的瞬间有可能其他的生产者线程先拿到了锁进行了生产，所以队列又经历了一个从不满到满的过程。

另外我们基于的是多生产者和消费者的情况，所以采用的是notifyAll而不是notify。

二、Lock和Condition的await / signal

需要2个Condition，分别用来唤醒生产者和消费者。另外唤醒是采用的是 signalAll, 不要使用成了notify。

三、阻塞队列

阻塞队列的内部实现就是 Lock + Condition，所以不需要再另外加锁。
