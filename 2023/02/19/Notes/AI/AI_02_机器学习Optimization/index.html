Local min和saddle Point 问题

Loss 无法变低的原因？

优化的目标是找到最优解 也就是 Loss 函数的最小值，loss 不能再下降了，loss下降的主要方法是通过判断grident，但是当 Grident接近于零，Loss 不能下降时，优化就不能继续了。Loss 不能下降的原因有2个：

  1. Local min
  2. saddle Point

Patch大小对Optimization的影响？

并行计算情况下 smallBatch 并没有时间上的优势，反而由于update次数多，epoch的时间较长。但是在同样的 Model情况下，优化的效果更好，主要原因是Noisy update能够有效的避免 Local mini 的情况，

Momentum 优化Small Gradient

Momentum不仅仅考虑当前的 gradient的方向，还会考虑前一次(或者说是过去所有Gradient的总和) 的情况。

比如下面这种情况，即使在最后一个点的 gradient 方向是向左的，但是由于之前的 movement 都是向右的，所以这个点上，仍然有可能向右移动，从而找到更好的 Loss

Traning Stuck

训练卡住的时候到底是不是因为 gradient 足够小了呢，不一定，比如下面的情况，Loss函数是一个山谷型的， gradient在山谷左右来回震荡，Loss 不能再小了，但是grident 仍然很高

LearningRate 对优化的影响

不同的L earringRate 对

当gradient 在某一个方向上的变化非常的剧烈，那么 Learning Rate就要设定小一点，如果在某个方向想变化的非常平缓，那么Leaning Rate就可以设定的大一点。

如何设定自适应的LearingRate?

Root Mean Square

RMSProp

可以动态调节当前 Grident对 LearningRate的影响

目前最常用的策略是：RMSProp + Momentum

建立与时间的关系

LearningRateDecay：Learning Rate 随时间变小

Warm Up：LearningRate 先变大再变小。一开始数据不精准的， 需要学习多笔资料后，统计才能够精准起来，所以 刚开始的Learning rate 可以小点，有一个探索的过程。后面在逐渐增大减小。

小节

Loss 函数

通过讲分类引出cross-entropy这种新的loss function，然后就可以讲loss function对训练的影响啦，告诉我们train不起来可能是loss function的问题~

批次標準化 (Batch Normalization)

训练数据不同纬度的Range,比如 w2的范围远远超过 w1 的话 也会对 Loss 函数产生影响。

Feature Normalization

做了处理后，所有的输入参数的均值是 0，方差为 1。所有维度的数值都在0 上下。

Batch Normalization

当我们对每一层的输出都做Normalization 的时候，相当于将所有的维度进行了关联，形成了一个 network 。当数据特别大的时候，只考虑一个Batch的情况，这就是Batch Normalization
